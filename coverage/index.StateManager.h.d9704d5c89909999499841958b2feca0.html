25 * @class StateManager
26 * @brief Manages a stack of GUI states for the application
27 *
28 * The StateManager class implements a stack-based state management system
29 * that allows for clean transitions between different GUI screens. It maintains
30 * a stack of states where only the top state is active and receives events.
31 *
32 * Key features:
33 * - Stack-based state management (LIFO - Last In, First Out)
34 * - Automatic state lifecycle management (onEnter/onExit)
35 * - Event delegation to the active state
36 * - Support for state overlays (multiple states can be active)
37 * - Memory management using smart pointers
38 *
39 * Common usage patterns:
40 * - pushState(): Add a new state on top (e.g., opening a settings menu)
41 * - popState(): Remove the top state (e.g., closing a dialog)
42 * - changeState(): Replace the current state (e.g., going from menu to game)
43 *
44 * Example:
45 * @code
46 * StateManager manager(window);
47 * manager.pushState(std::make_unique<MainMenuState>(manager));
48 * // In main loop:
49 * manager.handleEvent(event);
50 * manager.update(deltaTime);
51 * manager.render();
52 * @endcode
53 */
